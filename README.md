# FRR MCP Server

An [MCP (Model Context Protocol)](https://modelcontextprotocol.io/) server for managing FRR (Free Range Routing) routers. Enables AI assistants like Claude to configure and query FRR devices through natural language.

## Features

- Get all interface statuses and IP addresses
- Configure interface IP addresses, descriptions, and shutdown state
- View routing tables
- View OSPF neighbor adjacencies
- Add static routes
- Ping from the device

## Requirements

- Python 3.10+
- [Containerlab](https://containerlab.dev/) (for the lab topology)
- FRR router reachable via SSH

## Setup

### 1. Clone the repository

```bash
git clone <your-repo-url>
cd <repo-name>
```

### 2. Create and activate a virtual environment

```bash
python3 -m venv venv
source venv/bin/activate
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. Deploy the lab topology

```bash
sudo clab deploy -t topology.yml
```

### 5. Start the MCP server

```bash
bash startup.sh
```

Or directly:

```bash
python server.py
```

## Connecting to Claude

Add the following to your Claude MCP configuration (e.g. `claude_desktop_config.json`):

```json
{
  "mcpServers": {
    "frr-mcp": {
      "command": "python",
      "args": ["/path/to/server.py"]
    }
  }
}
```

## Notes

- The `clab-mcp-lab/` directory is generated by Containerlab at runtime and is not tracked in version control.
- Never commit your `venv/` directory — recreate it from `requirements.txt`.
