# FRR MCP Server

An MCP (Model Context Protocol server for managing FRR (Free Range Routing) routers. Enables AI assistants like Claude to configure and query FRR devices through natural language.

## Features 7 tools:

Read/Diagnostic
- get_device_info — hostname, FRR version, uptime
- get_interfaces — all interfaces with status and IP addresses
- get_routes — full routing table, or lookup a specific prefix
- get_ospf_neighbors — OSPF neighbor adjacencies
- ping_device — reachability test from the router (up to 20 pings)

Configuration
- configure_interface — assign IP, set description, shut/no shut an interface
- add_static_route — add a static route by prefix, prefix length, and next hop

## Requirements

- Python 3.10+
- [Containerlab](https://containerlab.dev/) (for the lab topology)
- FRR router reachable via SSH
- an AI Agent - Claude Desktop is used here.

## Setup

### 1. Clone the repository

```bash
git clone <your-repo-url>
cd <repo-name>
```

### 2. Create and activate a virtual environment

```bash
python3 -m venv venv
source venv/bin/activate
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. Deploy the lab topology

```bash
sudo clab deploy -t topology.yml
```

### 5. Start the MCP server

```bash
bash startup.sh
```

Or directly:

```bash
python server.py
```

## Connecting to Claude

Add the following to your Claude MCP configuration (e.g. `claude_desktop_config.json`):

```json
{
  "mcpServers": {
    "frr-mcp": {
      "command": "python",
      "args": ["/path/to/server.py"]
    }
  }
}
```

## Project Structure

```
.
├── server.py        # MCP server implementation
├── topology.yml     # Containerlab lab topology
├── startup.sh       # Server startup script
├── requirements.txt # Python dependencies
└── README.md
```

## Notes

- The `clab-mcp-lab/` directory is generated by Containerlab at runtime and is not tracked in version control.
- Never commit your `venv/` directory — recreate it from `requirements.txt`.
